# AGENTS.md

## Cursor Cloud specific instructions

### Overview

SvelteForge Admin is a SvelteKit admin dashboard using Svelte 5, Tailwind CSS v4, Supabase Auth (via `@supabase/ssr`), and Drizzle ORM with PostgreSQL. See `CLAUDE.md` for full command reference.

**Note:** The `README.md` and `CLAUDE.md` reference SQLite/better-sqlite3/custom-auth, but the codebase has been migrated to **PostgreSQL + Supabase Auth**. The actual database driver is `postgres` (postgres-js), the schema uses `pgTable`/`uuid`/`timestamp`, and auth flows through `@supabase/ssr`. Always trust the source code over stale docs.

### Services

| Service | How to start | Port | Notes |
|---|---|---|---|
| Supabase (Postgres + Auth) | `supabase start` (requires Docker) | 54321 (API), 54322 (DB) | Must run first. Provides PostgreSQL and Auth. |
| SvelteKit dev server | `pnpm dev` | 5173 | Requires `.env` with Supabase keys. |

### Environment setup

1. Docker must be running before `supabase start`.
2. If Supabase is not initialized, run `supabase init` first (creates `supabase/config.toml`).
3. After `supabase start`, get keys via `supabase status -o json` and populate `.env` (see `.env.example`).
4. Run `pnpm db:push` to apply the Drizzle schema, then `pnpm db:seed` for default app_settings.
5. The first user registered through the UI gets the `admin` role.

### Known issues

- The dashboard page (`src/routes/(app)/+page.server.ts`) has a bug where JavaScript `Date` objects are passed directly to `sql` template literals. The `postgres-js` driver does not serialize them automatically, causing a 500 error on the main dashboard. Other pages (Users, Content, Analytics, etc.) work fine. Fix: convert Date objects to ISO strings with `.toISOString()` before passing to `sql` templates.
- `pnpm lint` has 3 pre-existing errors and 7 warnings (in generated shadcn-svelte components and docs pages).
- `pnpm check` has 1 pre-existing TS error (`requestSubmit` not found on `HTMLElement` in `app-sidebar.svelte`).

### Testing

- Unit tests use PGlite (in-memory PostgreSQL) — see `test-utils.ts`. No external DB required for `pnpm test`.
- All 33 unit tests pass out of the box.
- E2E tests require `npx playwright install` before `pnpm test:e2e`.

### Gotchas

- `pnpm.onlyBuiltDependencies` in `package.json` limits native builds to `sharp` and `esbuild`. The legacy `better-sqlite3` dep may show an "ignored build scripts" warning — this is harmless since the app now uses `postgres-js`.
- The `supabase/` directory is auto-generated by `supabase init` and gitignored. Each agent session needs to re-initialize it.
